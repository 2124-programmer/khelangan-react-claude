# Android Release Signing & SHA-1 (D-05 / F5)

> **This project uses the managed Expo workflow — `android/` is gitignored and regenerated by
> `expo prebuild` / EAS Build.** So **Path A (EAS-managed credentials) is the authoritative path.**
> The `android/app/build.gradle` release-signing block (reads creds from Gradle properties / env,
> falls back to debug) helps only for one-off local builds against the *existing* `android/` folder
> and will be overwritten on the next prebuild — to make it permanent across regeneration, move it
> into a config plugin. **No keystore or password is committed to source** either way.

You must create the keystore once and provide the credentials. Pick the path matching how you ship.

---

## Path A — EAS Build (recommended; matches `eas.json` `production`)

EAS manages the upload keystore for you in the cloud. Nothing to wire into Gradle.

```bash
# Generate (first build) or inspect the EAS-managed keystore:
eas credentials -p android        # choose: Keystore → Set up / Download

# Print the SHA-1 / SHA-256 you must register with Google:
eas credentials -p android        # choose: Keystore → "View key hashes"
```

Register the printed **SHA-1** in:
- Google Play Console → Setup → App integrity (for upload + app-signing keys), and
- Google Cloud Console → your OAuth client (needed for Google Sign-In, F1/PL-04).

> Production profile already builds an `app-bundle` (`eas.json`). Keep app-signing by Google Play enabled.

---

## Path B — Local release build (`npm run build:apk:local`)

### 1. Create the upload keystore (once — keep it safe & backed up; losing it blocks updates)

```bash
keytool -genkeypair -v \
  -keystore scoreadda-upload.keystore \
  -alias scoreadda-upload \
  -keyalg RSA -keysize 2048 -validity 10000
# You'll be prompted for store + key passwords and a distinguished name.
```

Store the file **outside** the repo (e.g. `~/keystores/scoreadda-upload.keystore`).

### 2. Provide credentials WITHOUT committing them

Add to `~/.gradle/gradle.properties` (user-global, never in the repo):

```properties
SCOREADDA_UPLOAD_STORE_FILE=/absolute/path/to/scoreadda-upload.keystore
SCOREADDA_UPLOAD_STORE_PASSWORD=********
SCOREADDA_UPLOAD_KEY_ALIAS=scoreadda-upload
SCOREADDA_UPLOAD_KEY_PASSWORD=********
```

(Alternatively export the same names as environment variables, or pass `-P` flags.)
When these are present the release build is signed with this keystore; when absent it falls back to
the debug keystore so dev builds still work.

### 3. Get the SHA-1 to register with Google

```bash
keytool -list -v -keystore /absolute/path/to/scoreadda-upload.keystore -alias scoreadda-upload
# Copy the "SHA1:" line → register in Google Play Console + Google Cloud OAuth client.
```

## Checklist
- [ ] Keystore created and **backed up** in a secure location (not in git).
- [ ] Credentials in `~/.gradle/gradle.properties` (Path B) **or** managed by EAS (Path A).
- [ ] SHA-1 registered in Google Play Console (app integrity).
- [ ] SHA-1 registered in Google Cloud OAuth client (unblocks Google Sign-In, F1).
- [ ] `.gitignore` excludes `*.keystore` / `*.jks` (verify before first release commit).
